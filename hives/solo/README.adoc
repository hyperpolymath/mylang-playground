== Overview

Solo is a systems programming language designed as the foundation for the Next-Gen Languages initiative. It provides low-level control with high-level safety guarantees.

=== Key Features

* **Memory Safety**: Compile-time ownership tracking
* **Zero-Cost Abstractions**: No runtime overhead for safety
* **Bare Metal Support**: Direct hardware access
* **Interoperability**: FFI with C and assembly
* **Formal Verification**: Integration with proof systems

== Language Design

=== Type System

[source,solo]
----
-- Linear types for resource management
type File = linear {
    handle: FileHandle,
    mode: FileMode,
}

-- Must be closed exactly once
fn close(file: File) -> Result<(), Error>:
    sys_close(file.handle)
----

=== Memory Model

[source,solo]
----
-- Region-based allocation
region stack:
    let buffer = [u8; 1024]@stack
    process(buffer)
    -- buffer deallocated at region exit

-- Explicit heap allocation
let data = Box.new@heap(MyStruct { ... })
----

=== Systems Programming

[source,solo]
----
-- Direct memory access
fn mmio_write(addr: *mut u32, value: u32) -> ():
    volatile_store(addr, value)

-- Inline assembly
fn disable_interrupts() -> ():
    asm!("cli")
----

== Getting Started

=== Prerequisites

* LLVM 17+
* Rust toolchain (for compiler)
* Guix or Nix (package management)

=== Installation

[source,bash]
----
# Via Guix
guix install solo

# Or build from source
git clone https://github.com/hyperpolymath/solo
cd solo && cargo build --release
----

=== Hello World

[source,solo]
----
-- hello.solo
module Main

import Runtime.IO (print)

fn main() -> ():
    print("Hello from Solo!")
----

[source,bash]
----
solo build hello.solo
./hello
----
