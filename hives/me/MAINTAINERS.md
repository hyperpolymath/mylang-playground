## Decision-Making Process

### Consensus-Seeking
- Prefer consensus among Perimeter 1 & 2
- Use GitHub Discussions for major decisions
- Document reasoning in decision records (docs/decisions/)

### Conflict Resolution
1. Discussion and compromise attempt
2. If deadlocked, Lead Maintainer has final say
3. Dissenting opinions documented
4. Re-evaluation after 3-6 months if requested

### RFC Process
For major changes:
1. Submit RFC (Request for Comments) as GitHub Discussion
2. Community feedback period (2 weeks minimum)
3. Revision based on feedback
4. Final decision by Perimeter 1
5. Implementation by anyone

## Time Commitment Expectations

**Perimeter 1:**
- ~5-10 hours/week (flexible, no burnout)
- Timely security response (48 hours)
- Monthly planning meetings (optional)

**Perimeter 2:**
- ~2-5 hours/week (as available)
- No required meetings
- Contribute at your own pace

**Perimeter 3:**
- No expectations! Contribute as you wish
- One PR, one comment, all valued equally

## Stepping Down

Maintainers may step down at any time:
- No justification needed
- Life happens, priorities change
- Can return anytime (Perimeter 2 or 3)
- Emeritus status with honor and gratitude

Simply notify other maintainers via email or GitHub Discussion.

## Adding New Maintainers

### For Perimeter 2:
1. Existing Perimeter 1 or 2 member nominates
2. Candidate contacted privately
3. If accepts, announced publicly
4. Access granted within 1 week

### For Perimeter 1:
1. Existing Perimeter 1 member nominates
2. Private discussion among Perimeter 1 (unanimous required)
3. Candidate contacted
4. If accepts, public announcement
5. 1-month shadow period before full access

## Values Alignment

All maintainers commit to:
- **Emotional safety**: Prioritize contributor well-being
- **Transparency**: Document decisions, share reasoning
- **Inclusivity**: Welcome all backgrounds and skill levels
- **Reversibility**: Embrace experiments, learn from failures
- **Political autonomy**: No surveillance, respect privacy
- **Offline-first**: Software works without network
- **Community governance**: TPCF, Code of Conduct

## Succession Planning

If Lead Maintainer becomes unavailable:
1. Remaining Perimeter 1 members elect temporary lead
2. Permanent lead elected within 3 months
3. If no Perimeter 1 members remain, project archived with dignity
